01 · La fricción

El análisis empezaba cuando el producto terminaba.

La idea original era descargar la operación, exportarla a Excel y analizarla afuera. El dato estaba disponible, pero la decisión seguía dependiendo de trabajo manual y contexto disperso.

El punto de partida

Un dump completo servía como respaldo y materia prima. No resolvía qué mirar primero ni qué cambio merecía atención.

El giro de producto

Si el sistema ya tenía los datos, podía analizarlos donde nacían. El dumper pasó de extracción a inteligencia aplicada.

02 · Decisiones

La ingeniería se ordenó alrededor de tres decisiones.

01

Analizar dentro del producto

Las señales se convierten en una salida principal. La exportación deja de ser el destino del flujo.

02

Separar hechos de señales

Primero se construye una base operacional confiable; después se calculan agregados, patrones y anomalías.

03

Hacer visible la operación

Workers, checkpoints, snapshots y observabilidad permiten saber qué está fresco, qué falló y qué debe continuar.

03 · Sistema

De la API externa a una señal lista para decidir.

El flujo desacopla sincronización, almacenamiento, cálculo y lectura de negocio. Cada etapa responde a una restricción distinta y puede observarse por separado.

BaseFacts y snapshots
