Case study

Silver Usage Report

Un flujo privacy-first para reportar uso local agregado de Codex sin pedir credenciales ni instalar agentes permanentes.

Ver repo Mas proyectos
Rol Producto, backend, seguridad, collector
Dominio Telemetry, privacy, reporting
Stack FastAPI, HTMX, PostgreSQL, PowerShell

El desafio era generar confianza antes de recibir datos.

Contexto

El usuario necesitaba reportar uso agregado de Codex a Silver. El riesgo era cruzar una linea sensible: prompts, respuestas, codigo, logs o secretos no podian salir de la maquina.

Construccion

Arme una sesion privada que genera un collector PowerShell auditable. El script lee sesiones locales, muestra preview y solo envia agregados con confirmacion explicita.

Decision clave

No hay daemon, instalador ni tracking permanente. Es un flujo one-shot con token privado, firma HMAC y validaciones recursivas contra campos sensibles.

Privacidad como requisito de arquitectura, no como nota al pie.

El producto esta disenado para que Silver reciba evidencia agregada, no contenido. La confianza se apoya en preview local, submit firmado, ventana temporal corta y rechazo server-side de datos peligrosos.

  • Local-first collector PowerShell lee `%USERPROFILE%\.codex\sessions` y agrega por dia y modelo, sin empaquetar contenido sensible.
  • Signed submission HMAC, token privado y timestamp con ventana de 5 minutos para reducir replay y abuso.
  • Server-side trust model Hash de token, validacion Pydantic, filtros recursivos y costos calculados con tabla controlada por servidor.
