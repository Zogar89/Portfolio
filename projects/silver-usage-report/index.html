Silver Usage Report

Una plataforma creada en una tarde para que candidatos puedan enviar datos de uso de IA de forma facil, universal y segura.

Ver repo Mas proyectos
Rol Producto, backend, seguridad, collector
Dominio Telemetry, privacy, reporting
Stack FastAPI, HTMX, PostgreSQL, PowerShell

Nacio de un desafio real de Silverdev.

El pedido

Gabriel, de Silverdev, conto que las empresas pedian data sobre cuanto usaban IA los candidatos. La solucion tenia que ser facil de enviar y universal.

La respuesta

En una tarde arme una plataforma web con sesion privada, collector PowerShell, preview local y envio firmado de datos agregados.

El limite

La herramienta no podia transformarse en spyware ni en tracker permanente. Tenia que probar uso de IA sin exponer prompts, respuestas, codigo o secretos.

Facil de enviar, dificil de abusar.

El producto resuelve una tension concreta: las empresas quieren senales de uso de IA, pero el candidato no deberia entregar contenido privado. Por eso el flujo es one-shot, auditable y basado en agregados.

  • Local-first collector PowerShell lee `%USERPROFILE%\.codex\sessions` y agrega por dia y modelo, sin empaquetar contenido sensible.
  • Signed submission HMAC, token privado y timestamp con ventana de 5 minutos para reducir replay y abuso.
  • Server-side trust model Hash de token, validacion Pydantic, filtros recursivos y costos calculados con tabla controlada por servidor.

Stack y señales tecnicas.

Python 3.13 FastAPI Jinja2 HTMX Pydantic SQLAlchemy Alembic PostgreSQL SQLite fallback PowerShell HMAC Privacy by design pytest Docker Compose

Resultado: un requerimiento sensible convertido en un producto verificable, auditable y facil de ejecutar.
