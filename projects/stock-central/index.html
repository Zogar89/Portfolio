Case study

Stock Central

Un catalogo estatico para convertir stock disperso de filamentos 3D en una consulta clara, rapida y accionable.

Ver demo Ver repo
Rol Producto, ingesta, frontend, CI/CD
Dominio Catalogo, scraping, data quality
Stack Python, Svelte, Vite, GitHub Actions

El problema no era mostrar productos. Era publicar disponibilidad sin mentir.

Contexto

Los makers comparan filamentos entre proveedores, marcas, materiales, colores y formatos. La informacion vive distribuida, cambia seguido y no siempre llega limpia.

Construccion

Arme una ingesta en Python que normaliza catalogos y genera JSON estatico para una UI Svelte servida por GitHub Pages. No hay backend en runtime.

Decision clave

El producto no intenta vender ni cerrar pedidos. Es una herramienta de consulta y cotizacion local, con copy que obliga a confirmar stock y precio final.

Arquitectura pensada para operar barato y fallar con cuidado.

La separacion entre captura, UI y miniaturas permite publicar cambios de datos sin arrastrar todo el sitio. Los workflows de GitHub Actions cuidan que una fuente rota no degrade el catalogo publico.

  • Data quality guardrails Bloqueos ante catalogo vacio, caidas sospechosas de conteo o proveedor con stock inconsistente.
  • Static site generation JSON versionado, frontend estatico y despliegue por GitHub Pages sin servidores que mantener.
  • Human-in-the-loop Curacion de imagenes y miniaturas con reglas para no romper asociaciones de color, producto o marca.

Stack y señales tecnicas.

Python 3.12 Svelte 5 Vite GitHub Actions GitHub Pages BeautifulSoup httpx lxml Pillow pytest Static data pipeline CI/CD

Resultado: un producto util sin sobredisenar infraestructura, cuidando datos, UX y publicacion automatizada.
