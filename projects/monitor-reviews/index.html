Monitor de Reviews

Una primera aproximacion fuerte a la API de Mercado Libre para centralizar reviews y trackear como evolucionan los productos.

Dominio Reviews, reputation, operations

Mercado Libre tiene estadisticas, pero no resuelve esta vista.

La grieta

Un vendedor puede ver estadisticas, pero no tiene un lugar centralizado para mirar reviews de sus publicaciones y seguir que tal van sus productos.

La busqueda

El proyecto explora esa grieta: conectar cuentas, traer publicaciones y reviews, ordenar la informacion y convertirla en una superficie de seguimiento.

La dificultad

La API de MeLi no entrega todo listo. Hay polling, reviews compartidas por catalogo o familia y necesidad de deduplicar sin mezclar productos.

Una base para cerrar el hueco entre reputacion y producto.

Monitor de Reviews no intenta reemplazar toda la operacion de un seller. Se concentra en una pregunta precisa: que dicen las reviews y como evoluciona la percepcion de cada producto publicado.

  • Multi-account OAuth Usuarios con varias cuentas de Mercado Libre, tokens renovables y aislamiento por `request.user`.
  • Background synchronization Polling cada 24 horas, sync al login si esta stale y sync manual para compensar la falta de webhook dedicado.
  • Centralized review tracking Reviews, publicaciones, rating oficial y actividad historica reunidos para seguir la salud de los productos.
  • API learning surface Primer terreno fuerte para entender OAuth, SocialToken, endpoints, paginacion, modelos propios y limites reales de MeLi.

Stack y señales tecnicas.

Django 5 PostgreSQL 16 Django-Q2 Redis django-allauth OAuth2 Mercado Libre API HTMX Alpine.js Tailwind CSS DaisyUI Gunicorn WhiteNoise Sentry-ready

Resultado: informacion cualitativa convertida en una herramienta de seguimiento, sin esconder la complejidad de la plataforma externa.
