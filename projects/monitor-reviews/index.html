Case study

Monitor de Reviews

Un centro operativo para convertir reviews de Mercado Libre en prioridad, contexto y decisiones accionables.

Ver portfolio GitHub
Rol Backend, producto, sync, seguridad
Dominio Reviews, reputation, operations
Stack Django, PostgreSQL, Redis, HTMX

El producto no necesitaba mas graficos. Necesitaba una cola de atencion.

Contexto

Las reviews afectan reputacion, conversion y confianza. Pero Mercado Libre no entrega un webhook dedicado y las reviews pueden compartirse por catalogo o familia de productos.

Construccion

Arme una base Django con OAuth, multiples cuentas, sync automatico, cliente centralizado de MeLi, modelos de reviews y dashboard de atencion prioritaria.

Decision clave

La prioridad no la decide una IA opaca. El score es deterministico y auditable. La IA etiqueta y enriquece contexto en background.

Rescatar una base y llevarla a producto operable.

La direccion tecnica fue fortalecer fundamentos: settings reales, Gunicorn, Django-Q2, Redis, cliente API unico, filtros por usuario, seguridad OAuth y docs canonicas.

  • Multi-account OAuth Usuarios con varias cuentas de Mercado Libre, tokens renovables y aislamiento por `request.user`.
  • Background synchronization Polling cada 24 horas, sync al login si esta stale y sync manual para mantener datos frescos.
  • Attention units Agrupacion por familia, grupo funcional o item para no duplicar alertas ni mezclar competidores.
  • AI-assisted enrichment Sentimiento, etiquetas y patrones se calculan en background. La UI renderiza evidencia ya procesada.
